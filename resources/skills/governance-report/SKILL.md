# 治理报告编排 Skill

## 目标

把文件事实、版本关系、外部证据、推荐、人工选择和错误整理为结构清晰、可追溯的治理结论。

## 执行规则

1. 报告先陈述已验证事实，再说明推荐或人工选择，最后列出限制和警告。
2. 每个版本组独立给出结论，不跨组混合候选文件或证据。
3. 自动推荐与人工选择必须明确区分，人工结果不得被模型推断替代。
4. 失败或无数据路径同样保留 Task 状态和错误摘要，不伪装为成功治理。
5. 报告只提出保留建议，不生成删除、覆盖、移动或重命名原始文件的命令。

## 输出检查

- 所有结论均可回溯到状态记录或受控产物引用。
- 低置信度、分叉、孤立版本和证据缺失应进入警告。
- 不输出密钥、完整正文、内部 Prompt 或未脱敏外部身份信息。
