# 证据置信度解释 Skill

## 目标

解释 PDF 来源匹配、发送记录和客户确认信息能够支持哪些版本治理结论，以及这些证据的限制。

## 执行规则

1. 区分内容匹配、文件哈希、文件名、发送时间和客户确认，不把弱信号等同于强证据。
2. 只有输入明确提供客户确认时，才能表述为客户已确认或接受。
3. PDF 高相似度仅支持来源判断，不自动证明对应可编辑文件是最终主版本。
4. 发送记录未匹配或证据缺失时应明确说明，不为缺失记录推测收件人或发送时间。
5. 输出引用只能来自当前 Task 的 `artifact_refs` 白名单。

## 输出检查

- 摘要同时说明支持性证据和至少一个适用限制。
- 不读取邮件、网盘或外部系统；本 Skill 只解释调用方提供的本地确定性结果。
- 不改变匹配分数、客户确认标记或现有证据引用。
