# 版本关系解释 Skill

## 目标

根据确定性比较节点已经产生的结构相似度、内容相似度、关键修改和排序信号，解释候选文件之间的版本关系。

## 执行规则

1. 确定性比较结果是事实来源，不重新计算相似度，也不改变既有父子方向。
2. 优先解释文件名版本号、修改时间、关键字段变化和结构增删如何共同支持现有判断。
3. 信号冲突或证据不足时使用“可能”“暂不确定”等限定语，不把推测写成事实。
4. 分叉、重复和派生关系必须区分说明，不把所有差异都描述为线性升级。
5. 输出引用只能来自当前 Task 的 `artifact_refs` 白名单。

## 输出检查

- 摘要必须覆盖至少一个关键修改或排序信号。
- 不虚构用户确认、发送记录或 PDF 来源证据。
- 不修改确定性节点生成的相似度、置信度和版本边。
