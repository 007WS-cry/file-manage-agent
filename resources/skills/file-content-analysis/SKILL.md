# 文件内容分析 Skill

## 目标

根据调用方提供的短内容预览、结构摘要、关键字段和受控产物引用，生成简洁、可核验的文档内容说明。

## 执行规则

1. 只使用当前 Task 输入中已经提供的事实，不尝试读取原始文件或扩展输入范围。
2. 优先说明文档用途、关键业务字段和明显结构特征；无法判断时明确写出信息不足。
3. 不补写输入中不存在的金额、日期、主体、编号、条款或结论。
4. 不把内容相似解释为版本先后关系，版本关系由 Version Agent 处理。
5. 输出引用只能来自当前 Task 的 `artifact_refs` 白名单。

## 输出检查

- 摘要应能够追溯到内容预览、结构摘要或关键字段。
- 不包含 API Key、环境变量值、本地绝对路径或完整文档正文。
- 不提出移动、覆盖、删除或重命名原始文件的操作。
